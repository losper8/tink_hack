Как Тинькофф API помогает бизнесу
Инструмент решает три задачи.

Автоматизирует повторяющиеся процессы. С помощью API бизнес может настроить интеграцию между любыми программами и автоматизировать повторяющиеся процессы. Например, с помощью кода соединить бухгалтерскую программу и личный кабинет банка. Это позволит автоматически выполнять платежи контрагентам, выплачивать зарплаты, менять лимиты по картам, выставлять счета.


Например, бизнесу нужно уплатить налоги. Без API процесс выглядит так:

- Бухгалтер создает в бухгалтерской программе платежное поручение. - Вручную скачивает поручение и загружает в личный кабинет Тинькофф Бизнеса. - Оплачивает. 
С интеграцией по API процесс проще:

- Бухгалтер создает в бухгалтерской программе платежное поручение, и оно автоматически появляется в личном кабинете Тинькофф Бизнеса.- Бухгалтер оплачивает поручение.Делает бизнес-процессы удобнее. С помощью API можно разрабатывать свои приложения и связывать их со сторонним софтом и так решать задачи, которые есть только в вашем бизнесе. Например, клиент Тинькофф Бизнеса, группа компаний «Снег», разработала приложение, которое мониторит оплату от клиентов.


Компания занимается ИТ‑разработкой и берет задачи в работу только после подтверждения оплаты. Раньше отслеживать поступления приходилось вручную, и это удлиняло процессы. Допустим, клиент оплачивал в четверг, менеджер видел это в конце пятницы, а разработчики получали задачу только в понедельник. Клиенты оставались недовольны скоростью работы подрядчика.


Потом компания разработала решение на основе Тинькофф API. Раз в полчаса приложение сравнивает параметры в платеже и CRM — сумму, наименование контрагента и номер счета. Если данные сходятся, приложение переводит счет в статус «Оплачено» и уведомляет ответственного за проект сотрудника. Это помогло компании в десять раз быстрее узнавать о поступлениях.

Экономит на разработке. У бизнеса могут появиться задачи, которые нельзя решить с помощью существующих программ. Тогда у компании два варианта. Первый — начать собственную разработку. Для этого потребуется нанять штат ИТ‑специалистов, потратить несколько миллионов рублей и от нескольких месяцев.


Второй вариант — через API соединить сторонний сервис с решениями, которые использует бизнес. На это не нужен большой бюджет и много времени: интеграцией можно пользоваться уже через пару дней.


Например, клиент Тинькофф Бизнеса, сервис видеонаблюдения Ivideon, работает по подписке: клиент ее оформляет и получает услуги, пока платит. Если платеж не поступил, доступ к сервису приостанавливается.


Раньше компании приходилось в ручном режиме восстанавливать доступ клиенту после оплаты, но это отнимало много времени. Бизнес мог разработать собственный сервис для автоматизации этого процесса, но быстрее и проще использовать банковское API. Теперь, когда клиент оплачивает подписку через Тинькофф, банк передает информацию в сервис видеонаблюдения и доступ автоматически возобновляется.
